How I Built This Blog

Kelly Chan

Jan 09 2014


Keywords: HTML, CSS, Github


Knowledge Required

- HTML/CSS: to build the web pages

- Github Pages: to host the website


Process

Requirements Analysis -> Paper Prototype -> Architecture -> Coding -> Testing -> Hosting -> Documentation


1. Requirements Analysis

Functions Analysis:

---- categories with different pages

---- specific category with a list to display all posts of that category

---- specific post with a full page to display


Structure Analysis:

index.html (categories) -> (category).html (n category) -> (post).html (n posts)

index.html

|-------- art.html

|-------- tech.html

|-------- research.html

|-------- (gallery).html

|-------- (slides).html

|-------- life.html


2. Paper Prototype

To draw the prototype on a paper



3. Architecture

Folder Structure


HTML

To create the HTML templates, define the header and content for each portion. Set up the class and id for each page.

- index.html

- categroy.html

- yymmdd-Template_Name.html


CSS

To define CSS values in HTML templates (index.hmtl / category.html / yymmdd-Template_Name.html).


4. Coding & Tesing

Referred to the above design, coding the template pages (index.html / category.html / yymmdd-Template_Name.html) first.

Once finished and fine tune, then copied the structure to other categories (art/tech/research/life), modified the text value. Fine tune again.


5. Hosting

Now, upload all files to github repo, create a branch gh-pages, host the website on Github Pages.

Then test the links to see if all is in order.


6. Documentation

Finally, create the documents to mark down the project structure for the further reference.

Here, I wrote three documents: file_structure, css_specification, and how_to_post.


Notes

Time cost: 50% for coding and 50 % for testing and modification for the first released.

In fact, testing part is time-consuimg but deserved, we can find many little problems and fix them time and time again.
